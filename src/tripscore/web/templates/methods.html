Scoring model
  • Accessibility: proximity to transit layers and origin distance blend.
  • Weather comfort: rain/temperature comfort with configurable weights.
  • Preference match: tags (indoor/outdoor/culture/food/family/crowd_low).
  • Context: crowd and family context signals when available.
Degradation policy
  • Rate limits: targeted real-time queries back off under cooldown.
  • Missing data: components fall back to neutral rather than guessing.
  • Auditability: each result includes component reasons and data quality hints.
What to do if results look wrong
  • Verify the time window and starting point.
  • Check Data status for missing/unsupported datasets in your city.
  • Use Compare to test alternative scenarios (modes/weights).
